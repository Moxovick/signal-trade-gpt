# Мнение по текущему состоянию проекта `signal-trade-gpt-`

> Анализ архива `1WinRAR.zip`. Версия: 29.04.2026.

---

## TL;DR

Проект разделён на 3 части (бот / статический лендинг `web/` / Next.js платформа `web-platform/`) — это уже само по себе путаница: дубль кодовой базы, два разных представления одного и того же продукта, размазанная логика. **Дизайн в `web-platform/` уже жёлтый** (#f5c518), но плоский, без «вау» — он не близко к тому уровню анимации/деталей, который есть в `signal-trade-gpt-v3.html`. **Главное архитектурное несоответствие**: вся монетизация в коде сделана через подписки (FREE/PREMIUM/VIP/ELITE по $0/$29/$79), хотя по новой бизнес-модели подписок быть не должно — деньги идут с депозитов на PocketOption через партнёрку.

## Что не так — по пунктам

### 1. Дизайн (главная боль пользователя)

- В `web-platform/` уже есть **жёлтая палитра** (`#f5c518` accent, gold-gradient, glow), но визуально получился **тусклый flat-дашборд**: тёмный фон + блёклый сетка-overlay + плоские карточки. Никаких частиц, ни орбит, ни кастомного курсора, ни плавного перехода между секциями — то, что есть в v3.html, **не перенесено**.
- Иконки — это **emoji в строках** (`📊 📱 💎 🎯`). На лендинге продающего продукта это сразу сигналит «AI-сгенерёный сайт». Нужны нормальные SVG-иконки (Lucide Icons уже стоит в `package.json` — но нигде не используется!) или собственный icon-set.
- Шрифты подключены (Bebas / Manrope / JetBrains Mono), но используются хаотично: где-то `var(--font-bebas)`, где-то ничего, где-то `font-bold` без font-family.
- **Logo.tsx** — буквы «Z» и «K» в SVG (см. `M12 28L16 12H20L24 28H21L20 24H16L15 28H12Z`) с зелёной точкой; никак не связан с тематикой trading/AI/жёлтый. Логотип нужно переделать.
- Адмика, ЛК и лендинг **выглядят как три разных проекта**: лендинг — насыщенный с golden-gradient, ЛК — голый dark grid, админ — минималистичный list. Должна быть **одна дизайн-система**.

### 2. Бизнес-модель — критическое несоответствие

В коде сейчас:
- `SubscriptionPlan { free, premium, vip, elite }` + `Subscription` модель + цены $29/$79 на лендинге + `PromoCode` для триалов.
- В UI кнопки «Подключить Premium →» и «Подключить VIP →».
- Реферальная программа — **внутренняя**, считает % с **наших** же платежей пользователей (10%/15%/20% от подписок).

В вашей новой модели:
- **Подписок нет вообще.**
- Доступ к боту даётся только тем, кто либо зарегистрирован на PocketOption по нашей реф-ссылке, либо ввёл нам свой PocketOption trader ID и мы убедились, что он наш реферал.
- Деньги мы получаем от **PocketOption** как партнёр (revenue-share / CPA с FTD), а не от пользователей.
- «Уровни» (`OTC` / `Биржа` / `Elite`) разблокируются по сумме депозита лида на PocketOption — `$500 → tier 2`, `$2000 → tier 3` и т. д.
- Внутренняя реф-программа (юзер приглашает юзера) тоже остаётся, но она — про **plus-фичи в боте**, а не про % с подписок.

Это означает: модели `Subscription`, `Payment`, `PromoCode` **по сути не нужны** в текущем виде. Зато нужны новые: `PocketOptionAccount`, `Postback` (журнал), `BotFeature` / `BotPerk` / `Tier` (что и при каком депозите разблокируется).

### 3. PocketOption интеграция — её нет

В коде PocketOption упомянут только как ссылка `https://pocketoption.com` в .env бота. Никакой логики проверки, что пользователь — наш реферал, нет. Никакого учёта депозитов через postback. Админ-страница `/admin/deposits` есть, но это **ручная** верификация через `proofUrl` (загрузить скрин чека) — это плохо, лиды будут мухлевать.

**Что есть на самом деле у PocketOption** (после ресёрча):
- **Кабинет партнёра**: `https://affiliate.pocketoption.com`
- **Postbacks 2.0** — server-to-server callbacks; пять событий: `Registration`, `Email Confirmation`, `First Deposit (FTD)`, `Re-Deposit`, `Commission (RevShare daily)`. Передают `trader_id`, `click_id` (наш ID лида, который мы вшиваем в реф-ссылку), сумму, валюту.
- **CPA или RevShare** — выбираешь в кабинете; RevShare даёт 50–80% от прибыли брокера в зависимости от ранга (Regular/Premium/Influence/Ambassador, нужно от 3 / 50 / 50 / 200 FTD соответственно).
- **Публичного REST API для запросов «дай статистику»** — нет, всё через postback и CSV-экспорт из кабинета.
- Есть неофициальный реверс-API (`ChipaDevTeam/PocketOptionAPI`, Python, через SSID cookie + websocket) для **трейдерских** операций (свечи, баланс, открытие сделок) — это пригодится для real-time графиков в боте, но не для партнёрки.

**Вывод:** интеграция строится на **postback-листенере** (наш HTTPS-эндпоинт `/api/postback/po`, который PocketOption дёргает при событиях) + ручная сверка trader_id, который пользователь сам пришлёт в бот, со списком наших рефералов из CSV-экспорта.

### 4. Сигналы — фейк навсегда?

Сейчас бот генерирует чисто рандомные сигналы (`signal_generator.py`, направление 50/50, confidence random 73–96%). Для MVP это ок, но если хотим к графикам PocketOption привязать — нужен следующий шаг:
- Либо **технический индикатор-движок** (SMA/EMA/RSI cross на исторических свечах с PO через Chipa API).
- Либо **парсер сигналов из публичных Telegram-каналов** + переоформление в наш формат.
- Либо **ручной ввод сигналов аналитиком** (есть `SignalType { ai, expert, manual }` в схеме — уже заложено).

В админке `web-platform/src/app/admin/signals/page.tsx` есть только список — **нет формы создания вручную**. То есть админ не может прямо сейчас сам отправить сигнал.

### 5. Остальные мелочи (которые в сумме создают «AI-вайб»)

- В `page.tsx` смешаны украинский и русский: `OTC Угоди`, `Біржеві Угоди`, `Еліт Угоди` — а вокруг русские тексты. Так быть не должно, выбрать один язык интерфейса.
- В Prisma-схеме `prisma.config.ts` стоит `provider = "prisma-client"` (новый Prisma 7 generator) с `output = "../src/generated/prisma"` — это рабочий, но нестандартный вариант. Агент должен учитывать.
- `next-auth: 5.0.0-beta.31` — beta. Может ломаться. Плюс `next: 16.2.4` и `react: 19.2.4` — bleeding edge, многие туториалы из интернета **не подходят**, нужно читать `node_modules/next/dist/docs/` (как и просит `web-platform/AGENTS.md`).
- В `globals.css` цвет фона `#08081a` → почти чёрный с лёгким синим. Жёлтый акцент по такому фону смотрится «казино», а не «премиум AI». Нужен другой фон: либо тёплый чёрно-коричневый (`#0d0a06`), либо вообще **light-on-yellow** для определённых блоков (см. PRD).
- `bot/.env` лежит в архиве — судя по всему, **с реальным токеном**. Надо проверить и при необходимости отозвать токен через `@BotFather`. В git его быть не должно.
- Дисклеймер про «не финансовый совет» в коде нигде не видно (он только в `CLAUDE.md`/`PRD.md`). Юридически обязателен на лендинге и в боте.

## Главный вопрос, который надо решить ДО кода

**На каком плане у вас партнёрка PocketOption — RevShare или CPA?**
От этого зависит:
- Какие postback-события мы слушаем (CPA = только FTD; RevShare = ещё и Re-Deposit + Commission daily).
- Какие пороги депозитов разблокируют tier-ы (`$500`, `$2000` — это пороги, при которых **выгодно** нам, а не пользователю).
- Можно ли сделать **двухуровневую** систему (наш юзер → его рефералы тоже капают нам через sub-affiliate 5%).

Если ответа пока нет — в PRD заложу **RevShare** как дефолт (он гибче и приносит дольше).
